## Project Overview

Smart DNS Companion ‚Äî an AI-powered DNS filtering and parental control system. A custom DNS proxy intercepts queries, performs fast blacklist lookups, and logs activity to SQLite. A FastAPI dashboard exposes REST APIs for managing the blacklist, viewing query stats, and interacting with an AI agent (Nemotron via Ollama). Designed for edge deployment on NVIDIA Jetson Orin Nano.

## Running the Project

```bash
# Install dependencies
pip install -r dns-server/requirements.txt
pip install -r flttr/requirements.txt

# Start DNS server + dashboard (DNS on :53, dashboard on :8080)
cd dns-server && python main.py

# Test DNS
dig @127.0.0.1 -p 53 google.com

# Test API
curl http://localhost:8080/api/stats/overview
```

No test suite or linter is configured.

## Architecture

Two main modules, integrated at startup in `dns-server/main.py`:

### dns-server/ ‚Äî DNS proxy
- `main.py` ‚Äî entry point; loads `config.yaml`, starts DNS server + dashboard in parallel
- `server/dns_server.py` ‚Äî UDP server on port 53, parses queries with dnslib
- `server/filter.py` ‚Äî `DomainFilter`: in-memory set-based blacklist with subdomain matching; `reload()` refreshes from file
- `server/resolver.py` ‚Äî forwards allowed queries to upstream DNS (1.1.1.1)
- `server/blocker.py` ‚Äî builds block responses (0.0.0.0 or NXDOMAIN per config)
- `config.yaml` ‚Äî all runtime config (ports, upstream DNS, blocking mode, paths)
- `lists/blacklist.txt` ‚Äî domain blacklist, auto-regenerated by API

### flttr/ ‚Äî Dashboard & AI agent
- `app.py` ‚Äî FastAPI app factory
- `database.py` ‚Äî SQLite schema (domain_lists, query_log tables) + connection helpers
- `query_logger.py` ‚Äî background thread, queue-based DNS‚ÜíSQLite bridge
- `agent.py` ‚Äî `FlttrAgent`: sends commands to Nemotron via Ollama (`localhost:11434`), parses JSON responses, calls REST API
- `routes/` ‚Äî REST endpoints:
  - `lists.py` ‚Äî CRUD for blacklist domains (+ bulk add)
  - `queries.py` ‚Äî paginated/filtered query log
  - `stats.py` ‚Äî overview, hourly breakdown, top blocked
  - `system.py` ‚Äî health check

### Key design decisions
- **Blacklist-only** filtering (no whitelist)
- **No LLM on the DNS hot path** ‚Äî all filtering is fast set lookup; LLM is only used by the agent for command interpretation
- **No auth** ‚Äî assumes trusted local network
- Route modules use module-level globals (`db_path`, `domain_filter`, `blacklist_txt_path`) set by `app.py` at startup
- Dashboard runs as a daemon thread; DNS server blocks the main thread
- QueryLogger is optional ‚Äî DNS server works standalone without it
