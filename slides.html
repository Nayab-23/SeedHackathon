The Why

Kids deserve a safer internet.
Parents deserve simplicity.

The Problem

  • Parental controls are clunky, cloud-dependent, and easily bypassed
  • Static blocklists go stale within days and can't adapt to new threats
  • Most solutions require per-device agents that kids can uninstall

Our Solution

  • Network-level DNS filtering — every device is protected automatically
  • On-device AI (Nemotron via Ollama) interprets natural-language parental commands
  • Runs locally on a Jetson Orin Nano — no cloud, no subscriptions, full privacy

Architecture

Two Systems, One Device

dns-server/

UDP :53 — C-level network filtering
  • CORE DNS Server — intercepts all queries
  • FILTER DomainFilter — blacklist lookup
  • RESOLVE Upstream via Cloudflare 1.1.1.1
  • BLOCK Returns 0.0.0.0 for blocked domains
  • LOG QueryLogger → SQLite
SQLite

flttr/

REST :8080 — AI intelligence layer
  • API FastAPI Dashboard for parents
  • AGENT Command Interpreter (NLP)
  • LLM Ollama + Nemotron on-device

Decision

Branch: block or resolve upstream

→
💾

Log & Store

QueryLogger writes to SQLite in background

Blocked Path

Domain found in blacklist → returns 0.0.0.0 immediately. No upstream request made. Fast, private, and absolute.

Allowed Path

Domain not in blacklist → forwards to Cloudflare 1.1.1.1 for resolution. Response returned to client as normal.

Observability

Every query (blocked or allowed) is logged to SQLite with timestamp, domain, and decision. Feeds the dashboard.
